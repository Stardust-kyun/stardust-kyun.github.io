Desktop Icons

This post is to document how I added desktop icons to my AwesomeWM rice and some of the hiccups I ran into.

To start off, there are two file managers with built-in desktop icons: PCManFM and Nemo. I use Nemo and would recommend using it far more than PCManFM, mainly because Nemo uses gtk3 as opposed to gtk2 and is actively maintained. Nemo comes with the command nemo-desktop, more commonly known as Cinnamon's desktop -- however, it can be used on any window manager with a few tweaks.

When running nemo-desktop initially, desktop icons will not appear and need to be enabled. To enable them run the following command:

gsettings set org.nemo.desktop show-desktop-icons true

For ease of use I suggest adding nemo-desktop to your window manager's autostart script. Additionally, nemo-desktop may not start due to the following line in /usr/share/applications/nemo-autostart.desktop, so you may need to remove it:

OnlyShowIn=X-Cinnamon;

Unlike most window managers, AwesomeWM treats clients like nemo-desktop as any other client. Because of this nemo-desktop will "occupy" tags, by default be confined to a single tag, and not be visible on others. To fix this I used the following hacks:

screen.connect_signal("request::desktop_decoration", function(s)
	awful.tag({ "1", "2", "3", "4", "5", "6", " " }, s, awful.layout.layouts[1])
end)
ruled.client.append_rule {
    id = "desktop",
    rule_any = {
        class = {
        "Nemo-desktop"
        }
    },
    properties = { sticky = true, tag = " " }
}

To avoid a tag appearing "occupied" I created another tag called " " exclusively for nemo-desktop. When using a taglist without a background or icons, like mine, it's unnoticeable. A rule is also added to make nemo-desktop sticky and therefore visible on all desktops.

Lastly, your wallpaper may be hidden by nemo-desktop since it covers the root window. There are two options to fix this:

  • Set your wallpaper through nemo. This is fine for a single rice setup but can make it more annoying to change your wallpaper.
  • Make nemo-desktop transparent by making this element transparent in your gtk theme:
    • notebook stack:only-child {
    background: transparent;
}

    The only other thing to keep in mind is that nemo-desktop uses its own root menu, meaning that you won't be able to use your own root menu. This isn't a problem for me since I like the root menu and it integrates well with Nemo, but it may be different for you.

    It's really disappointing to me that most window managers don't really support desktop icons since they make a standard rice much closer to a desktop environment. Hopefully this post is comprehensive enough that it can help with adding them to your own rice.
